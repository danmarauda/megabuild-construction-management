# Crush Agent Configuration

## Essential Commands
- **Dev server**: `npm run dev`
- **Build**: `npm run build`
- **Lint**: `npm run lint`
- **Preview**: `npm run preview`

## Code Style Guidelines
- Use TypeScript with strict typing
- Follow existing import organization patterns
- Component naming: PascalCase with clear descriptive names
- File naming: kebab-case for pages/components, PascalCase for components
- Use functional components with React hooks
- Error handling: Try-catch blocks with meaningful error messages
- CSS: Tailwind classes preferred, minimal custom CSS

## Testing
- No specific test framework configured
- No single test running command available

## Project Structure
- Pages in `src/pages/`
- Components in `src/components/`
- Types in `src/types/`
- Data mocks in `src/data/`

## Libraries Used
- React + React DOM
- React Router DOM
- HeroUI React
- TailwindCSS
- Recharts for data visualization
- Date-fns for date manipulation
- Framer Motion for animations
